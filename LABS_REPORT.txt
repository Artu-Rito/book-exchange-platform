### Задание 2.2: Схема авторизации на двух токенах
**Статус:** ✅ Выполнено

#### Access Token:
- Время жизни: 30 минут
- Тип: JWT (HS256)
- Содержит: user_id, role, exp

#### Refresh Token:
- Время жизни: 7 дней
- Тип: JWT (HS256)
- Хранится в БД с возможностью отзыва

**Файлы:**
- `backend/app/core/config.py` - Настройки SECRET_KEY, ACCESS_TOKEN_EXPIRE_MINUTES, REFRESH_TOKEN_EXPIRE_DAYS
- `backend/app/core/security.py` - Функции create_access_token, create_refresh_token, verify_token

### Задание 2.3: Серверная часть (FastAPI)
**Статус:** ✅ Выполнено

#### Эндпоинты аутентификации:
- `POST /api/v1/auth/register` - Регистрация нового пользователя
- `POST /api/v1/auth/login` - Вход (возвращает access + refresh токены)
- `POST /api/v1/auth/refresh` - Обновление access токена
- `POST /api/v1/auth/logout` - Выход (отзыв refresh токена)
- `GET /api/v1/auth/me` - Получение текущего пользователя

**Файлы:**

- `backend/app/models/user.py` (строки 64-78) - Модель RefreshToken

#### Проверка access token:
```python
async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: Session = Depends(get_db)
) -> User:
    """Получение текущего пользователя из токена"""
```

**Файл:** `backend/app/api/deps.py` - Dependencies для проверки токена и прав

#### Хранение и отзыв refresh token:
- Refresh токены хранятся в таблице `refresh_tokens`
- Поля: token, user_id, expires_at, is_revoked, created_ip
- При logout: is_revoked = True
